TEANCUM and HTML SEMANTIC ELEMENTS

SEMANTIC STRUCTURE

<HEADER>

Introduces new content and helps to organize your page. There can be multiple <header> elements on a page. Do NOT confuse with <head> or headings (<h1>, etc)!

Show Me!

<MAIN>

Organizes the primary content on a page. In the context of proper web development symantics, there should only be one <main> element per page. It should be unique to the page and not be a derivative of any other element (except, of course, <body>).

Show Me!

<FOOTER>

Typically has information about the company or website, copyright information, contact information, and more. There can be more than one <footer> on a page.

Show Me!

<NAV>

The <nav> element is used to block together a set of links. Not every link needs to be in a <nav> element, but it is typical if there is more than one link.

Show Me!

<ASIDE>

Related but separate information from the main content of the page. It is literally put to the side of the page.

Show Me!

<SECTION>

W3 says, "Thematic grouping of content, typically with a heading"

Show Me!

<DIV>

Just as the name implies, it is simply a division or section of a web page. The <div> tag seems to be mostly used for CSS formatting.
